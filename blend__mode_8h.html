◆ TuiBlendMode

enum TuiBlendMode

How the to blend the foreground and background colors with the channels of a TuiAtlas when drawing batch data.

Enumerator
TUI_BLEND_INVALID 

The blend mode is invalid.

This blend mode is returned when an error occurs, and is also used to specify that no blend mode is defined.

TUI_BLEND_NORMAL 

No color blending occurs and the foreground and background colors are ignored.

TUI_BLEND_FG_RED 

Blend mode that blends the glyph foreground color with the red color component of the atlas texture.

TUI_BLEND_FG_GREEN 

Blend mode that blends the glyph foreground color with the green color component of the atlas texture.

TUI_BLEND_FG_BLUE 

Blend mode that blends the glyph foreground color with the blue color component of the atlas texture.

TUI_BLEND_FG_ALPHA 

Blend mode that blends the atlas texture color based on the alpha channel to mix the foreground and background color, with an alpha value of 0 blending only the foreground color.

TUI_BLEND_BG_RED 

Blend mode that blends the glyph background color with the red color component of the glyph texture.

TUI_BLEND_BG_GREEN 

Blend mode that blends the glyph background color with the red color component of the glyph texture.

TUI_BLEND_BG_BLUE 

Blend mode that blends the glyph background color with the red color component of the glyph texture.

TUI_BLEND_BG_ALPHA 

Blend mode that blends the atlas texture color based on the alpha channel to mix the foreground and background color, with an alpha value of 0 blending only the background color.

TUI_BLEND_FIRST 

The first valid blend mode value.

TUI_BLEND_LAST 

The last valid blend mode value.
