TuiInstance Struct Reference

TUI rendering context. TuiPalette, TuiGlyphAtlas, and TuiPanel objects are all attached to a TuiInstance. You must create the TuiInstance before these other objects, and destroy it last. These objects should only be used on a single thread. TuiInstance create functions are always backend specific. This object is opaque, and its members are not meant to be accessed directly unless implementing a backend. Use functions functions from instance.h or opengl33.h instead.
