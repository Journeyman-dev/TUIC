TuiImage tuiImageClone ( TuiImage  image)

Create a deep copy of a TuiImage.

Parameters
imageThe TuiImage.
Returns
The cloned TuiImage. TUI_NULL is returned if an error occurs.
Errors:
This function can have the error TUI_ERROR_NULL_IMAGE. The first error that occurs will cause the function to immediatly return.
Requirements:
This function can be called freely, even if TUIC is not currently initialized.
Thread safety:
This function can be called safely on any thread. However, it is important to manipulate and use each TuiImage on only one thread at a time to ensure safe memory access.

Create a TuiImage with a raw pixel array.

Parameters
texture_formatThe format of the image (its channel count).
widthThe width of the image in pixels.
heightThe height of the image in pixels.
pixel_dataThe pixel data array.
copy_dataIf the pixel data should be deeply copied instead of shallow coppied.
Returns
The created TuiImage. TUI_NULL is returned if an error occurs.
Errors:
Possible errors in order are TUI_ERROR_INVALID_CHANNEL_COUNT and TUI_ERROR_INVALID_IMAGE_DIMENSIONS. The first error that occurs will cause the function to immediatly return. Also, an inccorectly sized or allocated pixels array may cause undefined behaviour or a fatal crash without an error.
Requirements:
This function can be called freely, even if TUIC is not currently initialized.
Pointer lifetime:
The returned TuiImage must be destroyed using the function tuiImageDestroy().
Thread safety:
This function can be called safely on any thread at any time.

◆ tuiImageGetPixels()

uint8_t* tuiImageGetPixels ( TuiImage  image)

Get a pointer to the raw pixels of a TuiImage.

Parameters
imageThe TuiImage.
Returns
A pointer to the pixel array of the image.
Errors:
This function can have the error TUI_ERROR_NULL_IMAGE. The first error that occurs will cause the function to immediatly return.
Requirements:
This function can be called freely, even if TUIC is not currently initialized.
Pointer lifetime:
The returned pixel array should never be freed, and will be destroyed when the image it was taken from is destroyed. Modifying the contents of the array will affect the image, and is not thread safe with other TuiImage functions on the same object.
Thread safety:
This function can be called safely on any thread. However, it is important to manipulate and use each TuiImage on only one thread at a time to ensure safe memory access.
